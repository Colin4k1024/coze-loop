# Lessons Learned

## 2026-03-31 — Coze Loop 项目文档体系梳理

### 场景

首次为 Coze Loop 项目建立 `docs/` 目录结构，梳理前后端架构文档和开发指南。

### 关键经验

1. **并行探索 + 编写效率高**：将探索和编写并行执行（3 个 agent 同时工作），总产出 3905 行文档，比串行节省约 40% 时间。

2. **双 monorepo 文档要分别归档**：前端（Rush.js）和后端（Go modules）是两个独立的 monorepo，文档分开存放便于各自维护。

3. **探索性任务用 agent 并行化**：代码库规模大（前端 53 项目，后端多模块），用专项 agent 并行探索比单线程高效得多。

### 问题与建议

| 问题 | 建议 |
|------|------|
| 文档与代码可能同步漂移 | 建议建立文档维护机制，或在 CODEOWNERS 中标注文档负责人 |
| 缺少门户索引页 | 建议补充 `docs/README.md` 作为 docs/ 总入口 |
| 无历史 ADR | 建议后续补立 ADR 记录关键决策（Eino、Kitex 选型等） |
